# Backlog Structure

- `feedback.md` — пожелания, UX/feature идеи
- `bugs.md` — известные баги и статус
- `fixes.md` — журнал внедрённых фиксов

Рекомендуемый поток:
1. Сначала создаём GitHub Issue по шаблону.
2. Дублируем краткую запись в соответствующий markdown-файл (для быстрого обзора в репо).
